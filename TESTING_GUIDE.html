Test Setup — Load Demo Data

Before You Start Any Testing

/admin/settings
In Firebase Console → Authentication → Users, create 4 test accounts with Email/Password: admin@test.com, receptionist@test.com, pharmacist@test.com, doctor@test.com.
In Firestore → userRoles collection, create a document for each user (Document ID = user's UID) with field role set to the appropriate value: admin, receptionist, pharmacist, doctor.
Each user document has a single field: role: "admin" (etc.)
Log in as admin@test.com → Navigate to /admin/settings → Click "Load Fresh Demo" (or "Seed Demo Data").
Green success card appears listing seeded items: 20 patients, invoices, cards, pharmacy bills, appointments, and queue tokens — all fully linked.
Open multiple browser windows or tabs to test different roles simultaneously (use incognito windows for separate sessions).
Demo data creates a realistic dataset. All records are cross-linked by phone number — so patient records, billing, and pharmacy data are all connected and ready to explore.

Issue Tokens & Work Through the Full 5-Stage Flow

Token appears in Waiting list with sequential number. Print slip option available.
Issue 3 more tokens for different patients.
All 4 tokens visible in queue with correct order.
Click "Call Next Patient".
First token moves to "Called" state (amber). WhatsApp notification would fire to patient's phone (if Twilio configured).

Click "Done — Call Next".

Backup, Restore & CSV Exports

Download & Restore Backup

/admin/settings
Go to Settings → click "Download Backup".
A JSON file is downloaded to your computer. File name includes date/timestamp.
Open the downloaded JSON file in a text editor.
File contains data for all 12 Firestore collections: patients, invoices, pharmacyBills, inpatientCards, queueTokens, appointments, medicines, stockMovements, suppliers, blog, gallery, reviews.
Click "Reset Data" to clear all data. Then click "Restore from Backup" → upload the JSON file downloaded in step 1.
All records restored from the backup file. Patient list, invoices, and pharmacy data reappear in the system.
Check if the 30-day backup reminder is displayed in Settings.
Banner or alert showing when the last backup was downloaded and prompting if it was more than 30 days ago.
Never test Restore on production data without first downloading a fresh backup. Restore replaces all current data.

CSV Exports

From /admin/patients (or the Patients page for the receptionist), click "Export CSV".
A .csv file downloads with all patients: PAT-XXXX, name, phone, email, total spend, registered date.
From the In-Patient Cards page, click "Export CSV".
CSV file with all IPD/OPD card records: card number, patient ID, name, doctor, ward, created date, expiry, status.
From the Token Queue page, click "Export CSV".
CSV file with the day's queue tokens: token number, patient name, phone, purpose, status, timestamps.
Open each CSV in Excel or Google Sheets.
Data loads correctly with proper columns. No encoding errors for Kannada or special characters.
