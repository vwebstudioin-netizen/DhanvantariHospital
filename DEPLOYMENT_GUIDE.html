1Prerequisites

Install these before starting:

  • Node.js 18+ — nodejs.org
  • Git (optional but recommended)
  • Google account for Firebase
  • Twilio account (only if testing WhatsApp)

Project location: Clients/ABHospital/ inside your TemplateS folder.

2Firebase Setup

We need a Firebase project for authentication, database, and file storage.

2.1 Create Firebase Project

Go to console.firebase.google.com
Click "Add project" → Name it dhanvantari-hospital → Continue
Disable Google Analytics (not needed) → Create project
Wait for project to be created → Click Continue

2.2 Enable Authentication

In left sidebar: Build → Authentication
Click "Get started"
Under "Sign-in method" tab → Click Email/Password → Enable → Save

2.3 Create Firestore Database

In left sidebar: Build → Firestore Database
Click "Create database"
Choose "Start in test mode" → Next
Select asia-south1 (Mumbai) for India → Click Enable

2.4 Enable Storage

In left sidebar: Build → Storage
Click "Get started" → Start in test mode → Next → Done

2.5 Get Client Config Keys

Click the gear icon ⚙ (top left) → Project settings
Scroll to "Your apps" → Click Web icon (</>)
Register app as dhanvantari-hospital-web → Continue
Copy the firebaseConfig object — you'll need these values for .env.local
✅ No service account key needed. This project uses Firestore-based role management — the Firebase Admin SDK and service account JSON are not required.

5Create First Admin User

No service account key or script needed. Roles are stored directly in Firestore. Just 3 steps:

5.1 Create the admin user in Firebase Auth

Firebase Console → Authentication → Users → Add user
Email: admin@dhanvantarihospital.com, Password: Admin@12345 → Add user
Copy the User UID shown (e.g. abc123xyz...)

5.2 Set the admin role in Firestore

Firebase Console → Firestore Database → Start collection
Collection ID: userRoles → Next
Document ID: paste the UID from step 5.1
Add these fields:
Field: role — Type: string — Value: admin
Field: email — Type: string — Value: admin@dhanvantarihospital.com
Click Save
That's it — no scripts, no service account key needed. Roles are just Firestore documents.

5.3 Test admin login

Go to http://localhost:3000/login
Login with admin@dhanvantarihospital.com / Admin@12345
Should redirect to /admin with full sidebar

13Setup Twilio WhatsApp (Optional)

Skip this for initial testing. Do this when the client wants live WhatsApp messages.

13.1 Create Twilio account

Sign up at twilio.com
Go to Console → copy Account SID and Auth Token
Paste into .env.local: TWILIO_ACCOUNT_SID and TWILIO_AUTH_TOKEN

13.2 Enable WhatsApp Sandbox (for testing)

Twilio Console → Messaging → Try it Out → Send a WhatsApp message
Follow instructions: send "join [code]" from your phone to the Twilio sandbox number
Sandbox number is: +1 415 523 8886 — this is your TWILIO_WHATSAPP_FROM
Restart dev server → Issue a token and call next → Check if WhatsApp message arrives

13.3 Test WhatsApp Bot

Set webhook URL in Twilio: https://your-domain.com/api/whatsapp/webhook
For local testing, use ngrok: ngrok http 3000 → copy the HTTPS URL
Paste in Twilio: Messaging → Sandbox → "When a message comes in" → your ngrok URL + /api/whatsapp/webhook
Send "APPOINTMENT" or "HELP" to the sandbox number → should get an auto-reply

14Full Test Checklist

Use this before handing over to the client:

Authentication & Roles

  • Admin login → redirects to /admin with full sidebar
  • Pharmacist login → redirects to /admin/pharmacy with pharmacy-only sidebar
  • Receptionist login → redirects to /desk with Token Queue link
  • Pharmacist cannot access /admin (redirected)
  • Receptionist cannot access /admin (redirected)
  • Receptionist cannot access /admin/pharmacy (redirected)
  • Role assignment via /admin/roles works
  • Logout redirects to /login

Public Website

  • Homepage loads with hospital branding from env variables
  • Services, Doctors, About, Contact pages load
  • Patient portal login works
  • Book appointment form works

Queue System

  • Issue token — appears in waiting list
  • Call next — token moves to "Now Serving"
  • TV display updates in real-time
  • Audio chime plays on TV display when token changes
  • Patient can check queue status at /queue
  • Token slip prints correctly

Desk App

  • Create in-patient card — expiry date auto-set to +14 days
  • Discharge patient — removed from active list
  • Print patient card — 85×54mm layout
  • Create invoice with multiple line items
  • Discount applies correctly
  • Invoice prints correctly
  • Invoice appears in /desk/bills history
  • Festive wishes campaign saves to Firestore

Pharmacy

  • Add a medicine — appears in catalog
  • Edit medicine — updates correctly
  • Add stock — stock count increases, movement logged
  • Dispense medicine — stock count decreases
  • Low stock alert appears on dashboard when stock ≤ reorder level
  • Add supplier — appears in list
  • Reports: stock status and expiry tracking show data

WhatsApp (if configured)

  • Token called → WhatsApp notification sent to patient's phone
  • In-patient card created → card details sent via WhatsApp
  • Festive wishes → bulk WhatsApp sent
  • Bot: send "APPOINTMENT" → auto-reply received
  • Bot: send "HELP" → menu received

✓ All done!

If all items above pass, the system is ready for the client. Remember to update the branding in .env.local with the actual hospital name, phone, address, and UPI details before handing over.
